# Anorha Web App

## Register

product

## Users

Resellers managing inventory, listings, sales channels, campaigns, billing, and teams from a desktop browser. They are often tired, impatient, and working through a backlog, so the interface must stay calm, direct, and easy to scan.

## Product Purpose

Anorha helps a seller turn an item into a complete listing, publish it across connected channels, keep inventory and listings in sync, and use Sprout to plan sales or liquidation campaigns. The web app extends the mobile workflow onto a larger screen without becoming a separate product or an operations-heavy admin tool.

## Brand Personality

Calm, capable, and friendly. The product should feel like a practical selling partner that has already done most of the work, not software that asks the seller to configure everything.

## Anti-references

Avoid generic SaaS dashboards, dense enterprise administration, nested card grids, decorative gradients, oversized metrics, glassmorphism, loud inactive states, and unfamiliar controls. Do not make the web app feel like an older beige companion to the mobile app.

## Design Principles

1. Default to simple and reveal complexity only when it is relevant.
2. Present work for confirmation instead of presenting blank forms.
3. Keep the normal case quiet and make exceptions or stale states obvious.
4. Preserve momentum with plain language, visible next actions, and forgiving exits.
5. Use the mobile app's Profile, Campaign Chat, and Add Product surfaces as the cross-platform source of truth.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Maintain keyboard navigation, visible focus states, semantic controls, sufficient text and status contrast, non-color status cues, 44px minimum touch targets where practical, and reduced-motion support.
